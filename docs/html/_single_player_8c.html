double evaluate ( Board *  board)

Evaluates a Move using the Board.

Using the number of current pieces and current kings, calculates how good the situation for a player after performing a Move. The formula that is being used: (white_pieces - black_pieces) * (white_kings * 0.5 + black_kings * 0.5) indicates how the evaluation is calibrated. The best thing that can happen in a Move is conquering a piece, if no conquest is possible than we'll check if we can promote a Soldier to an Officer. These parameters can be adjusted as pleased.

Parameters
boardThe board to be evaluated.
Returns
A number indicating the evaluation

Chooses a Move to be played using the Minimax algorithm.

Minimax Algorithm: the Minimax algorithm creates a game tree based on all of the possible moves a player can make. Every level of the tree, indicated by the depth, contains the children of the depth + 1, meaning every Move the opposite user can make after the user has chosen one of the moves. Once the tree has been generated, it recursively traverses the tree adding a value to every move, this value depends on how useful the Move is for a user, for example if it can conquer a piece or make a piece become officer. The higher the value, the better it is for the maximizing_player. Then, it chooses between Moves of the same level comparing these values, if the turn that is being checked is the maximizing_player's one, chooses the highest value, if not it chooses the lowest, since it is the worst decision to make for the opponent. Finally, it copies the best Move into the move so that we can use it to perform that move.

Parameters
moveThe Move that is being checked.
depthHow many turns the AI is checking to choose the Move, the higher this number is, the higher the computing power is needed to search the Move. The computational complexity is O(b^m), where b is the number of legal moves at each point and m is the maximum depth of the tree.
maximizing_playerThe player who is playing and who has to get the best Move.
boardThe board which we are the moves simulating on.
Returns
The best Move the AI can make.
